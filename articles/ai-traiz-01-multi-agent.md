## フック

英語をやり直したくて、AI に「英語コーチになって」と頼む——ここまでは誰でもやります。
でも1つのチャットに全部やらせると、すぐ破綻しました。カリキュラムを考えさせ、教材を作らせ、会話相手をさせ、採点もさせる。役割がぶつかって、文脈が膨らんで、「今日何をやればいいか」が毎回ぶれる。

そこで方針を変えて、**役割ごとに別々のエージェントへ分業**させたら、最終的に6体になりました。この記事はその設計の話です。

## 背景：なぜ作ったのか

自分の英語はやり直しレベル（レベル診断で A2 / TOEIC 300 相当）。独学が続かない理由は、いつもモチベーションではなく「毎日の"次の一手"が決まっていないこと」でした。市販アプリは教材はくれるけど、**自分専用のカリキュラム設計と、毎日サボりを取り立てる伴走**がない。

そこを AI で埋められないか、というのが出発点です。

## 設計判断：1つのAIに丸投げしない

最初の失敗が、まさに「1つのチャットに全部」でした。問題は3つ：

1. **文脈の肥大** — 教材も会話も採点も同じ会話に積むと、すぐ長くなって精度が落ちる。
2. **役割の衝突** — 「優しく励ます」と「厳しく採点する」を同じ人格にやらせると、どっちも中途半端になる。
3. **責任の所在が消える** — カリキュラムを変えたのは誰の判断か、が追えなくなる。

人間のコーチングサービスも、実は分業しています。カリキュラムを設計する人、教材を作る人、レッスンする人、評価する人。**それを AI で再現する**、と考えたら設計がすっきりしました。

## 6体のエージェント

| 役割 | やること | 駆動 |
|---|---|---|
| Consultant | カリキュラム設計・週次レビュー | Claude Code |
| Coach | 朝晩の伴走・サボり取り立て | 音声（Gemini Live / 読み上げ） |
| Native Speaker | 英会話・ドリルの相手 | Gemini Live |
| Material Curator | 日次の教材を自動生成 | Gemini 2.5 Pro |
| Evaluator | レッスンを8軸で採点 | Claude Code |
| Career Mapper | 英語力と市場価値のマップ | Claude Code |

判断の拠り所として、「流暢なビジネス英語」を8軸に分解したルーブリックと、学習手法のカタログ（各手法にエビデンスを付けたもの）を別ファイルに持たせています。エージェントはここを参照して判断するので、**"なぜその練習なのか"が毎回たどれる**。

## アーキテクチャ：GitHub が「唯一の真実」

```
iPhone PWA ──(音声レッスン)── Gemini Live
    │ commit / sync
    ▼
GitHub リポジトリ（唯一の真実＝学習台帳）
    ▲ pull / push
    │
Mac 常駐（launchd で自走）
  Consultant / Coach / Native Speaker / Curator / Evaluator / Career Mapper
```

キモは、**学習者の状態（プロフィール・カリキュラム・レッスン記録・採点）を全部 GitHub に置いた**こと。DBもSaaSも使わず、リポジトリが台帳です。

こうすると、

- 各エージェントは「リポジトリを読んで → 判断して → 変更をコミット」で完結する。
- 誰がいつ何を変えたかが git 履歴に全部残る（＝説明可能）。
- iPhone のレッスン結果も commit で同期。端末とエージェントが同じ台帳を見る。

そして Mac の常駐ジョブ（launchd）が、朝は教材生成とコーチの檄、夜は採点とダッシュボード更新、日曜は週次レビュー……と**勝手に回る**。人間（＝私）はレッスンを受けるだけ。

## つまづき（次回予告）

……という設計は綺麗なのですが、立ち上げ初日にいきなり地獄を見ました。**iPhone のマイクが音を拾わない・エコーが乗る・AudioWorklet が動かない**。「多エージェントの英語コーチ」以前に、**音声が通らないと何も始まらない**。

次回は、この**音声疎通地獄**（iOS Safari のマイク、AEC、AudioWorklet 撤退）を丸ごと書きます。

## 学び（持ち帰り）

- AI に複合タスクをやらせるなら、**役割で割って別エージェントにする**と、精度も説明可能性も上がる。
- **状態を1か所（今回は git リポジトリ）に集約**すると、エージェント間の連携が「読む・書く」だけで済む。
- ルーブリックや手法カタログを**外出しファイル**にすると、判断の根拠が再利用でき、ぶれない。
