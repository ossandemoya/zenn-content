> 連載「AIに英語コーチを作らせてみた」第1回。英語は話せない、コードも書いたことがない——そんな"おっさんデモ屋"が、AIに自分専用の英語学習アプリを作らせていく記録です。

## まず、私のスペック

- **英語**：やり直しレベル。アプリのレベル診断で A2（TOEIC 300 相当）。会話が続かない。
- **プログラミング**：一行も書いたことがない。エラーメッセージを見ても意味が分からない。

この2つが「できない」人間が、「AIに自分専用の英語コーチを作らせる」と決めました。結論から言うと、最初は盛大に失敗しています。

## 1つのAIに丸投げしたら破綻した

はじめは単純に、AIに「英語コーチになって」と頼みました。カリキュラムを考えさせ、教材を作らせ、会話相手をさせ、採点もさせる。……全部いっぺんにやらせたら、すぐぐちゃぐちゃになりました。

- 会話が長くなるほど、返事がぼやける
- 「優しく励ます」と「厳しく採点する」を同じ相手にやらせると、どっちも中途半端
- 昨日と今日で言うことが違って、「今日は何をやればいいの?」が毎回ぶれる

コードは書けないので、自分で直しようがありません。そこで**AI自身に「これ、どう分ければうまくいく?」と相談しながら**、役割を1つずつ切り出していきました。

## 気づいたら6人になっていた

人間のコーチングも、実は分業しています。設計する人、教材を作る人、レッスンする人、評価する人。それをAIで割ってみたら、6人になりました。

| 役割 | やること | 中身 |
|---|---|---|
| Consultant | カリキュラム設計・週次レビュー | Claude Code |
| Coach | 朝晩の声かけ・サボり取り立て | 音声 |
| Native Speaker | 英会話・ドリルの相手 | 音声 |
| Material Curator | 日次の教材づくり | AI生成 |
| Evaluator | レッスンを8軸で採点 | Claude Code |
| Career Mapper | 英語力と市場価値のマップ | Claude Code |

面白いのは、**私がやったのは「どう分けるか」を決めただけ**で、中の実装はぜんぶAI任せということ。コードが書けなくても、「何を・どう分担させるか」なら、AIと相談しながら決められました。

## 記録は全部「ひとつの場所」に置いた（GitHub）

作ったものの一番のキモは、学習者（＝私）の状態——プロフィール、カリキュラム、毎日のレッスン記録、採点——を**全部 GitHub に置いた**ことです。データベースも専用サービスも使わず、リポジトリがそのまま「学習台帳」。

非コーダーの私には、これが地味に効きました。自分でコードを読めないぶん、**誰が・いつ・何を変えたかが履歴に全部残る**のが安心なんです。エージェントたちは「台帳を読む → 判断する → 変更を記録する」の繰り返しで動き、iPhone のレッスン結果もそこに同期される。あとは Mac が、朝は教材づくりと声かけ、夜は採点、と勝手に回してくれます。

## で、初日にいきなり地獄を見る（次回）

設計は綺麗だったのに、立ち上げ初日、**iPhone のマイクが音を拾わない**。エコーは乗るし、録音は残らない。エラーの意味も分からない私は、ひたすらAIに投げて直してもらう羽目になりました。次回はこの**音声疎通地獄**を丸ごと書きます。

## 今回の持ち帰り

- 英語もコードもできなくても、「何を作るか・どう分けるか」を決めれば、書く部分はAIに任せられる。
- 複数のことをAIにやらせるなら、1つに丸投げせず**役割で分ける**とうまくいく。
- 状態を**1か所（git）に集約**しておくと、コードを読めなくても「履歴で追える」という安心がある。
